CONCLUSIONS :::

Temperature Range ::
======================================================================================
There are two temperature ranges, HIGH and LOW. Set by Client;
Temp range HIGH = Set temp ranges from 26.5 degree celcius to 40 degree celcius;
Temp range LOW = Set temp ranges from 10 degree celcius to 37 degree celcius;

When we change the Temp Range from the client(Display), it sends the following command:
7e 07 10 bf 11 50 00 32 7e

We can see the status change in the Status Update Frame's 
15th Index byte and its bit # 2. (High = bit is 1. Low = bit is 0).



Heating Mode ::
===========================================================
There are three heating modes; REST, READY & READY-IN-REST; Set by client;

Command sent by the client(display) to change the Heating Mode ::
7e 07 10 bf 11 51 00 27 7e

We are getting change in the Status Update Frame's
10th index Byte. (Byte = 0x00 = Heating mode Ready; 0x01 = Heating mode Rest; 0x02 = Read-in-Rest(Yet to understand how it get into this mode)).

When we switch to Ready mode from Resting mode, Circulation also starts autometically.



Circulation in normal mode ::
===========================================================
When Heating Mode is Ready, circulation starts autometically;

If Jet1, Jet2 or Blower1 starts, circulation starts autometically even if the Heating Mode is Resting.
Circulation can not be controlled manually while in a normal mode.


Circulation in Priming mode ::
=========================================================
In Priming mode, Circulation is not synced with Jet1 or Jet2. It is operated separately.
Circulation On and Off command from the Client(Display)::
7e 07 10 bf 11 3d 00 2e 7e

We can see the status change in the Status Update Frame's 
18th Index byte and its bit # 1. (Circulation pump on = bit is 1; Circulation pump off = bit is 0).
Also 15th Index byte's(Related to heating) bit # 1 shows gets 1 for both the states of circulation pump(on or off), which does not happen in normal mode(it stays 00).
It may be indicating that there is no heating status and we are in a priming mode.
The details for these bits are not mantioned in the referance document as well. It may be indicating Priming mode operations.

When we go out of the priming mode to the normal mode, circulation autometically starts;

7e 07 10 bf 11 01 00 2b 7e	|	Command to switch to Normal Operation(Exit Priming Mode).



Normal operation ::
=========================================================
Ready Mode:
If set and measured temperature are same, circulation & temp icon(red thermometer), which also coordinates with heater relay, will stay off.
If set temp is higher than the measured temp, it will start circulation, after some time, for few seconds, will on heater relay and display "Heating" icon.
Then will turn off the heater relay and the icon will change to "Heat Waiting" icon. If temperature still doesn't match, it will once again start the relay and 
will change the icon to "Heating" icon once again. If temperature starts increasing and reaches to the set temp level, heater realy will turn off along with the "Heating" icon.
After some time circulation icon will turn off along with its relay.



Filtration Cycle ::
=========================================================

As soon as you enter into the filtration menu, client sends a "Settings Request", having size of 10 bytes and 5th byte having 0x22 indicating it
is a settings request and 6th byte having 0x01 indicating that it is a request for filter cycles.

Once you click the "Correct" button after making the changes, the client sends another frame having size of 15 bytes. Of which 5th byte is 0x23 indicating it
is a "Filter Cycles Message" 


In order to Setup the "Filtration Cycle" Settings, client need to send only the following command of 15 bytes :
7e 0d 10 bf 23 0d 00 00 0f 8f 0f 00 0f 14 7e

Where the 5th byte having 0x23 indicaes that it is a "Filter Cycles Message".
6th byte indicates Cycle 1's Start time Hour
7th byte indicates Cycle 1's Start time Minute
8th byte indicates Cycle 1's Duration Hours
9th byte indicates Cycle 1's Duration Minutes
10th byte's 0-6 bits indicates Cycle 2's Start time Hour
10th byte's 7th bit indicates Cycle 2 is enable or disable(0/1)
11th byte indicates Cycle 2's Start time Minute
12th byte indicates Cycle 2's Duration Hours
13th byte indicates Cycle 2's Duration Minutes


Set Time ::
===========================================================
After power on, machine give a message/notification to set the time once.

We can set the time manually using the following command :
7e 07 10 bf 21 [] [] 89 7e

6th index byte	|	Hour; In hour's byte, the MSB should be 1 for some reason.
7th index byte	|	Minute

0x01	|	Priming Mode
0x02	|	Fault/Error
0x03	|	Reminder/Message/Information

Byte Index 11 : Reminder/Message/Error Code in HEX.
0x00	|	Idle / No Error
0x13	|	In priming mode
0x28	|	Set Time
0x1E	|	Heater Too Hot

6th index byte - 0x03	|	11th index byte 0x28(indicating Message Code 40)

If there is an error on the display we can send the following command :
7e 07 10 bf 11 03 00 01 7e	|	Clear Notification.




Setup Number & DIP Switch status ::
==========================================================
In order to get these info, client needs to send an Information Request command i.e. 7E 08 0A BF 22 02 00 00 [crc] 7E; Type of message is 0x22[Settings request].
The next byte 0x02 means it is an Information Request.
The controller will provide information response. Ex.: 7e 1a 10 bf 24 64 e2 2b 00 42 50 36 30 31 33 47 32 01 f7 1f e5 eb 03 0a 13 00 d3 7e;
17th Index shows "Setup Numtber"
24th and 25th index shows "DIP Switch Position" where bit 0 of byte index 24 is for position 1 of the dip switch.
Type of message will be 0x24[Information Response]


Hold Mode ::
==========================================================
The command to toggle between normal and Hold mode is 7e 07 10 bf 11 3c 00 3b 7e;
In the Status message, the byte index 5 shows 0x05 if the machine is in the hold mode. 0x00 if is in normal working mode.


Cleanup Cycle ::
=========================================================
Command to set cleanup cycle : 7e 07 10 bf 27 03 02 93 7e - Message type 0x27[Set Preferance request]
Byte index 5 is 0x03 which indicates that it is a cleanup cycle request.
The next byte indicates minutes; Can be set from 0 to 8. 0 means off, 1 means 0.5 hours, 2 means 1 hours, 3 means 1.5 hours and so on; Maximum can be 4 hours;

When we send an preferance request to set the clean up cycle, we get following preferance response;
7e 17 ff af 26 00 87 00 01 01 03 00 00 01 00 00 00 00 00 00 00 00 00 cc 7e; Type of message is 0x26
Byte index 10 shows the minutes/time duration for clean up cycle. Hear, 0x03 shows 1.5 hours.

if clean up cycle is on or off can be checked from the byte index 24, bit 0-3 of the status message.
